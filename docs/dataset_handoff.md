The YOLO dataset is ready and follows the agreed interface exactly. The exported structure is `dataset/images/train`, `dataset/images/val`, `dataset/images/test`, `dataset/labels/train`, `dataset/labels/val`, `dataset/labels/test`, plus `dataset/data.yaml` and `dataset/metadata.csv`. Every image has a matching `.txt` label file with the same name in the corresponding labels folder. The only class is `0: tampered_region`. Tampered images have one YOLO-format box in the label file using `class_id center_x center_y width height`, with all box values normalized between 0 and 1. Clean/authentic images have empty `.txt` label files so YOLO can learn negative examples. The dataset was exported from SIDTD templates using the real/fake image folders and the SIDTD annotation JSONs. For fake images, I used the fake metadata to identify the tampered field, then recovered the field bounding box from the corresponding real annotation file. I also added a small padding around boxes so they are not too tight and include visual context around the modified field. Final dataset stats: 2222 total images and 2222 matching label files, with 1000 clean images and 1222 tampered images. Split: train has 1557 images, 697 clean and 860 tampered; validation has 339 images, 153 clean and 186 tampered; test has 326 images, 150 clean and 176 tampered. Label consistency checks passed: total images = total labels, non-empty labels = 1222, empty labels = 1000, and all 1000 clean label files are empty. I also did manual visual QA using a deterministic review set under `outputs/visual_checks/manual_step8/`, including `train_tampered_sheet.jpg`, `val_tampered_sheet.jpg`, `test_tampered_sheet.jpg`, `clean_sheet.jpg`, and `review_samples.csv`. QA result: 20/20 train tampered examples correct, 10/10 validation tampered examples correct, 10/10 test tampered examples correct, and 9/9 clean examples had no boxes. Overall, 40/40 tampered visual checks were correct or very close. You should now be able to train directly with `dataset/data.yaml`, for example: `yolo detect train model=yolov8n.pt data=dataset/data.yaml epochs=30 imgsz=640 batch=8`. At document level, we can classify a document as potentially forged if YOLO detects at least one `tampered_region` above the selected confidence threshold.
